# v1.0.0 release blocker checklist

Use this checklist before promoting an RC to a production-ready v1.0.0 release.

- Compose one-command startup passes with bundled MySQL and external MySQL.
- Password authentication uses BCrypt or Argon2 and does not log session identifiers.
- Datasource and engine-client secrets are encrypted at rest and masked in API responses.
- Backend tests run in CI; frontend lint, type-check, tests, and build run in CI.
- Database migrations are versioned, documented, and tested against backup/restore flows.
- Release workflow creates GitHub Release artifacts, SHA-256 checksums, SBOM, and image provenance.
- Changelog, security policy, and deployment templates point to this repository and its maintainers.
- All `release-blocker` issues in the `v1.0.0` milestone are closed.
